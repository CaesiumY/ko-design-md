# Security Policy

## Supported Versions

`main` 브랜치만 지원합니다. 본 프로젝트는 카탈로그 사이트 + 정적 콘텐츠가 주이며, 별도 LTS 라인을 두지 않습니다.

## Reporting a Vulnerability

보안 이슈는 **공개 이슈로 올리지 말아주세요.**

대신 본 리포의 [Security Advisories](../../security/advisories/new)를 통해 비공개로 보고해주세요.

- 회신 목표: 영업일 기준 72시간 이내 첫 응답
- 보고 채널은 단일 메인테이너([@CaesiumY](https://github.com/CaesiumY))에게 직접 전달됩니다.

## Out of scope

다음 항목은 본 정책의 보안 이슈로 다루지 않습니다.

- 카탈로그 콘텐츠(`services/*.md`)의 사실 오류 또는 출처 부정확성 — [GitHub Issues](../../issues)의 `entry-correction` 템플릿을 사용해주세요.
- 사이트 UI의 디자인/레이아웃 버그 — `bug-report` 템플릿을 사용해주세요.

## 브랜드 자산 삭제 요청 (Takedown)

`public/logos/` 안의 브랜드 자산에 대한 삭제 요청은 동일하게 [Security Advisories](../../security/advisories/new)를 통해 비공개로 보고해주세요. 권리자 확인을 거쳐 신속히 처리합니다. 자세한 정책은 [NOTICE](./NOTICE)를 참고하세요.
