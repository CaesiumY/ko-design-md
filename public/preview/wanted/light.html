05.

Do's & Don'ts — 다운스트림 LLM을 위한 가드레일

단일 강조색 · alpha-multiplied text · 그라디언트는 4가지 자리에만

Do

  • 화면당 단일 강조색을 유지한다 — primary CTA는 bg-brand 하나로 통일한다.
  • 텍스트 위계는 13단계 알파 스케일로 합성한다 — primary 88 / secondary 61 / tertiary 43 / disabled 28.
  • 카드는 1px border-subtle 헤어라인으로 구조를 만든다 — 그림자는 elevated surface에만.
  • radius는 8 또는 12를 디폴트로, pills는 full(9999)로 사용한다.
  • 본문은 body1(1.5)을, 산문은 body1-read(1.625)를 사용한다.
  • 버튼 라벨은 동사로 — "지원하기 / 저장하기 / 둘러보기".
  • 잡카드 우측 하단에는 채용보상금을 항상 fg-brand로 노출한다.
  • 아이콘은 24×24 / 2px / outline / currentColor 상속.
  • 포커스 링은 항상 visible — 2px blue-800 + 2px transparent offset.

Don't

  • 이모지를 product UI에 inline으로 사용하지 않는다 — monochrome SVG 또는 평면 일러스트.
  • 그라디언트를 CTA · 헤더 · 풀-블리드 표면에 적용하지 않는다 — 심볼 · 아바타 · 카드 썸네일 · hero banner 4자리에만.
  • 격식체(-습니다)나 단정형(-다)을 product 카피에 쓰지 않는다 — -요 / -어요가 표준.
  • ALL-CAPS, Title Case In Buttons를 쓰지 않는다 — sentence case 일관.
  • UI 라벨 끝에 마침표를 찍지 않는다.
  • 카드에 그림자를 적용하지 않는다 — 헤어라인 보더로 구조를 표현한다.
  • 6 · 10 · 14 · 18 · 22 같은 비-4의 배수 spacing/radius를 도입하지 않는다.
  • 텍스처 · 노이즈 · grain · glassy blur를 표면에 사용하지 않는다.
  • spring · bounce · parallax · slide 모션을 사용하지 않는다 — hover 100~150ms, page fade ~200ms.
  • gray-* 패밀리를 UI 표면 색으로 직접 사용하지 않는다 — neutral-* 패밀리가 워크호스.
